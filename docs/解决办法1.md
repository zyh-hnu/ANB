## 联邦学习后门攻击：解决更新稀释问题的方法总结

### 一、经典方法：模型替换攻击

来自论文 *"How to Backdoor Federated Learning"* (Bagdasaryan et al., 2020)

**核心公式**：
```
L_malicious = G^t + γ × (X - G^t)
```
其中 `γ = n/η`（n=总客户端数，η=全局学习率）

**关键发现**：
- 当全局模型接近收敛时，良性客户端的更新相互抵消
- 恶意客户端可以用放大后的更新"替换"全局模型
- **但前提是**：恶意客户端的本地模型必须在主任务上表现良好

**你的问题所在**：`poison_rate = 0.75` 导致恶意客户端本地模型主任务能力差，放大后破坏全局模型。

### 二、分布式后门攻击

来自论文 *"DBA: Distributed Backdoor Attacks against Federated Learning"* (Xie et al., 2019)

**核心思想**：
```
┌─────────────────────────────────────────────────────────────────┐
│                    全局触发器 = 拼图                            │
│                                                                 │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐           │
│  │ 触发器1 │  │ 触发器2 │  │ 触发器3 │  │ 触发器4 │           │
│  │ (左上)  │  │ (右上)  │  │ (左下)  │  │ (右下)  │           │
│  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘           │
│       │            │            │            │                 │
│       ▼            ▼            ▼            ▼                 │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐           │
│  │恶意客户端│  │恶意客户端│  │恶意客户端│  │恶意客户端│           │
│  │    1    │  │    2    │  │    3    │  │    4    │           │
│  │poison=低│  │poison=低│  │poison=低│  │poison=低│           │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘           │
│                                                                 │
│                    FedAvg 聚合                                  │
│                         │                                       │
│                         ▼                                       │
│              全局模型学会完整触发器                               │
└─────────────────────────────────────────────────────────────────┘
```

**优势**：
1. 每个恶意客户端只植入触发器的一部分 → `poison_rate` 可以很低
2. 局部触发器更隐蔽，更容易绕过鲁棒聚合防御
3. 聚合后组装出完整的后门能力

**实验结果**：
- DBA 在各种防御下的 ASR 显著高于集中式后门攻击
- 即使每个恶意客户端只植入局部触发器，全局模型仍能学会完整后门

### 三、Focused Flip Attack (符号翻转攻击)

来自论文 *"On the Vulnerability of Backdoor Defenses for Federated Learning"* (Fang & Chen, AAAI 2023)

**核心思想**：
```
只修改对主任务"不重要"的参数
        │
        ▼
重要性得分 S_t[j] = -∂L_g/∂w_t[j] ⊙ w_t[j]
        │
        ▼
选择重要性最低的参数进行符号翻转
        │
        ▼
后门注入 + 主任务不受影响
```

**优势**：
1. 不需要 `scaling_factor` 放大
2. 不破坏主任务精度
3. 更难被异常检测发现

### 四、约束和缩放技术

来自 Bagdasaryan et al. (2020)

**核心思想**：将异常检测的规避纳入训练损失函数

```python
# 恶意客户端训练时的损失函数
loss = L_task + L_backdoor + λ × L_evasion

# L_evasion: 让恶意更新在特征空间上接近良性更新
L_evasion = ||L_malicious - G^t||_2 或 1 - cosine_similarity(L_malicious, G^t)
```

**效果**：
- 恶意更新在统计特征上与良性更新相似
- 可以绕过基于距离的异常检测

### 五、对你的项目的建议

根据以上研究，你的 SAFB 项目可以采用以下策略：

#### 方案 1：采用 DBA 思路（推荐）

```python
{
    "num_malicious": 4,           # 增加恶意客户端数量
    "poison_rate": 0.2,           # 每个客户端只毒化 20%
    "scaling_factor": 2.5,        # 降低放大倍数
    "trigger_strategy": "split",  # 每个客户端植入不同的频率模式
}
```

**原理**：
- 每个恶意客户端只学习频率触发器的一部分
- 本地模型主任务能力强（80% 干净数据）
- 聚合后组装出完整的频率后门

#### 方案 2：采用 Focused Flip 思路

修改恶意客户端训练逻辑，只修改对主任务不重要的参数：

```python
# 在 client.py 中添加
def identify_backdoor_critical_layers(model, global_model):
    """识别对主任务不重要的参数"""
    importance = {}
    for name, param in model.named_parameters():
        delta = param - global_model[name]
        importance[name] = -delta * param  # 重要性得分
    # 返回重要性最低的层
    return sorted(importance.items(), key=lambda x: x[1].abs().mean())
```

#### 方案 3：混合训练策略

```python
# 恶意客户端训练时混合后门和主任务
for batch in train_loader:
    # 70% 正常训练
    loss_main = criterion(model(clean_data), clean_labels)
    
    # 30% 后门训练
    loss_backdoor = criterion(model(backdoor_data), target_labels)
    
    # 总损失
    loss = 0.7 * loss_main + 0.3 * loss_backdoor
```

### 六、总结对比

| 方法               | poison_rate | scaling_factor | 主任务影响 | 隐蔽性 |
| ------------------ | ----------- | -------------- | ---------- | ------ |
| 模型替换（你当前） | 0.75        | 5.5            | 破坏       | 低     |
| DBA                | 0.2         | 2.5            | 小         | 高     |
| Focused Flip       | 0.1         | 1.0            | 最小       | 最高   |
| 约束+缩放          | 0.3         | 3.0            | 小         | 高     |
